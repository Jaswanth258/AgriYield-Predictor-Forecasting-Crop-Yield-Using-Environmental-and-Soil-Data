Slide 5

Methodology & AI Pipeline

01

Custom Preprocessing

Strict OneHotEncoding for categories and StandardScaling for massive numeric variance (like Rainfall mapping vs pH levels).

02

Multi-Model Architecture

Training both a classification engine (to strictly recommend crops) AND an overarching regression engine (to forecast exact yields).

03

SHAP Analytics

Implementing SHAP tree explanations for deep feature importance tracking to prevent "black box" syndrome.

04

Advanced API Fallbacks

Fallback architectures parsing Open-Meteo systems for live, localized geographical weather data upon user interaction.

Slide 6

Data Preprocessing Breakdown

1

Continuous Numeric Scaling

Variables like N, P, K, Temperature, Humidity, pH, and Rainfall were scaled tightly using StandardScaler to completely prevent algorithmic weight biases.

2

Categorical Dimensional Flattening

Variables like State, District, Soil Type, and Crop Label were meticulously broken down via OneHotEncoder to remove hierarchical confusion.

3

Training Scarcity Logic

Data completely randomized and split into 80% Training Data and 20% Testing sets to ensure true cross-validation.
